# Photo Album Viewer

一个**本地优先**、面向大量照片目录的桌面浏览器。  
不要求导入图库，不改动原始文件，直接在你的真实文件夹结构上工作。

## 为什么还要做这个软件？

市场上确实有很多照片软件，但很多工具默认是“导入到数据库/云端后再管理”。

这个项目要解决的是另一类高频痛点：

1. 你的照片已经按年份/活动/设备散落在多层目录里，结构很深。  
2. 你只想**快速浏览、筛选、收藏**，不想先做一次“大迁移”。  
3. 你担心工具改动原文件（移动、重命名、覆盖）带来不可逆风险。  
4. 目录很大时，系统文件管理器预览慢、跳转成本高。

### 这个项目的产品定位

它不是“全能照片管理系统”，而是一个**高效、安全、零迁移成本**的本地相簿浏览器。

## 相比常见竞品，它的优势在哪里？

1. 目录即数据，无需导入
直接读取本地目录，避免建立和维护另一套图库。

2. 穿透多层文件夹，自动识别相簿
会把“最底层含图片的目录”作为相簿处理，减少手动层层点开的成本。

3. 默认非破坏式
核心流程是浏览、预览、收藏、定位到文件夹、复制原图文件引用；不做源文件改写。

4. 大目录可用性优先
缩略图缓存、批量预取、虚拟列表/网格等机制，面向大量图片场景优化浏览流畅度。

5. 本地优先与隐私友好
收藏和设置保存在本机 `userData`，不依赖云服务。

## 适合谁

- 照片都在本地磁盘/NAS 挂载目录，且目录层级复杂的用户
- 想快速看图、挑图、收藏，而不是先建立完整 DAM 流程的用户
- 对“原始文件安全”敏感，不希望工具替你改动文件结构的用户

## 不做什么（当前版本）

- 不提供云同步/在线账号体系
- 不做 AI 标注、人脸识别、自动分类训练
- 不做 RAW 开发、调色、非破坏编辑链路

## 核心功能

### 1) 文件夹与相簿浏览
- 选择任意根目录开始浏览
- 自动区分文件夹节点与相簿节点
- 文件夹/相簿搜索、排序（名称/数量/时间等）
- 面包屑导航与随机相簿跳转

### 2) 多标签工作流
- 在当前标签打开文件夹
- 在新标签打开文件夹
- 在新窗口打开文件夹
- 标签会话恢复与拖拽重排

### 3) 图片查看器（Image Viewer）
- 全屏查看、缩放、拖拽平移、旋转
- 键盘导航（上一张/下一张/关闭/全屏等）
- 在系统文件管理器中定位原图
- 复制原始图片文件引用（保留原文件名/后缀）
- 复制图片像素到剪贴板

### 4) 收藏与设置
- 收藏相簿、收藏单图
- 收藏数据持久化（本地）
- 深浅色主题、查看器偏好、缓存管理

## 支持格式

当前内置支持：
- `.jpg` / `.jpeg`
- `.png`
- `.gif`
- `.bmp`
- `.webp`

## 快速开始

### 环境要求
- Node.js 16+
- npm 7+

### 安装与运行
```bash
git clone <your-repo-url>
cd photo-album-viewer
npm install
npm start
```

### 构建
```bash
npm run build
```

## 常用命令

```bash
npm start              # 开发模式（Webpack dev server + Electron）
npm test               # 运行单元测试
npm run test:watch     # 监听模式测试
npm run build          # 生产构建与打包
```

## 快捷键（高频）

### 图片查看器
- `←` / `A`：上一张
- `→` / `D`：下一张
- `Esc` / `Backspace`：关闭查看器
- `+` / `=`：放大
- `-` / `_`：缩小
- `F`：切换全屏
- `O`：在文件管理器中显示当前图片
- `C`：收藏/取消收藏当前图片
- `Cmd/Ctrl + C`：复制原始图片文件引用
- `Cmd/Ctrl + Shift + C`：复制图片像素

### 文件夹/相簿页面
- `R`：随机跳转
- `Backspace`：返回上级（文件夹页）
- `H`：回到主页（相簿页）

## 数据与安全边界

- 默认只读照片目录
- 收藏数据：Electron `userData` 目录下的 `favorites.json`
- 缩略图缓存：Electron `userData` 目录下的 `thumbnail-cache`
- 清理缓存不会删除原始照片文件
